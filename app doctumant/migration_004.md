# notifications

Stores every in-app notification.

The client CANNOT create notifications.

Notifications are generated by future backend logic.

## Columns

| Column | Type | Description |
|---------|------|-------------|
| id | uuid | Notification ID |
| user_id | uuid | Receiver |
| actor_user_id | uuid | User who triggered the notification |
| type | enum | Notification Type |
| title | text | Notification title |
| body | text | Notification body |
| data | jsonb | Deep-link payload |
| is_read | bool | Read status |
| read_at | timestamptz | Read timestamp |
| created_at | timestamptz | Created |

## Flutter

Fetch notifications

```dart
final notifications = await supabase
    .from('notifications')
    .select()
    .order('created_at', ascending: false);
```

---

Unread notifications

```dart
final unread = await supabase
    .from('notifications')
    .select()
    .eq('is_read', false)
    .order('created_at');
```

---

Mark notification as read

```dart
await supabase
    .from('notifications')
    .update({

      'is_read': true,

      'read_at': DateTime.now().toIso8601String()

    })
    .eq('id', notificationId);
```

---

Delete notification

```dart
await supabase
    .from('notifications')
    .delete()
    .eq('id', notificationId);
```

# notification_preferences

One row per user.

Stores notification settings.

---

Columns

| Column | Description |
|---------|-------------|
| push_enabled | Master Push Switch |
| email_enabled | Master Email Switch |
| like_notifications | Like Alerts |
| match_notifications | Match Alerts |
| message_notifications | Message Alerts |
| call_notifications | Voice/Video Calls |
| profile_view_notifications | Profile View Alerts |
| marketing_notifications | Marketing Messages |

---

Get Preferences

```dart
final prefs = await supabase
    .from('notification_preferences')
    .select()
    .single();
```

---

Update Preferences

```dart
await supabase
    .from('notification_preferences')
    .update({

      'push_enabled': false,

      'message_notifications': true

    })
    .eq('user_id', user.id);
```

---

Create Preferences

Run only once after signup.

```dart
await supabase
    .from('notification_preferences')
    .insert({

      'user_id': user.id

    });
```

# Deep Link Payload

The `data` JSON column contains additional information for navigation.

Examples:

Match Notification

```json
{
  "match_id": "uuid"
}
```

Message Notification

```json
{
  "conversation_id": "uuid",
  "message_id": "uuid"
}
```

Profile View

```json
{
  "viewer_id": "uuid"
}
```

Flutter should inspect `type` and `data` together to determine where to navigate.
